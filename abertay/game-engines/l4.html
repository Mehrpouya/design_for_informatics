#Blueprint
- [Level Blueprint](https://docs.unrealengine.com/latest/INT/Engine/Blueprints/UserGuide/Types/LevelBlueprint/index.html) Whole level, Global things... They called this Kismet in UE3. You also use this for Matinee and Cinematics. - [Class Blueprint](https://docs.unrealengine.com/latest/INT/Engine/Blueprints/UserGuide/Types/ClassBlueprint/index.html) Each Actor can have one of these to make them alive. - [Data-only Blueprint](https://docs.unrealengine.com/latest/INT/Engine/Blueprints/UserGuide/Types/ClassBlueprint/index.html#data-onlyblueprint) This is now Enumeration & User defined structure. List of names, health and what player picked us, last checkpoint... - [Interface Blueprint](https://docs.unrealengine.com/latest/INT/Engine/Blueprints/UserGuide/Types/Interface/index.html) This a way to standardise your blue prints, for example to make sure all light blueprints have a function for switching on and off. - [Macro Blueprint](https://docs.unrealengine.com/latest/INT/Engine/Blueprints/UserGuide/Types/MacroLibrary/index.html) Think about it as utilities ... use in many places ... like a function to measure distance between two actors. They have inputs and outputs.
##What events or things happen inside a level?
- Player start - Death, Birth (Spawn, respawn) - Actors&Levels Ticks - Sensing (proximity, becoming view target) - Collisions, overlaps - Keyboard, Mouse, VR, mobile inputs...
